<task type="execute" id="3">
  <name>Task 3: Deploy to production</name>
  <read_first>
    - package.json
    - .vercel/project.json
  </read_first>
  <action>
1. **Verify build works locally first** (catch any production-only build errors before burning a Vercel build slot):
   ```bash
   bun run build
   ```
   Expect: `next build` completes with no errors. If it fails, STOP and fix before deploying.

2. **Deploy to production**:
   ```bash
   vercel --prod --yes
   ```
   Capture the deployment URL from the last line of stdout (typically `https://jtac-trainer-<hash>.vercel.app` and the alias `https://jtac-trainer.vercel.app`).

3. **Write the URL to disk** for Phase 6 to consume:
   ```bash
   vercel inspect --json $(vercel ls --json | jq -r '.[0].url') 2>/dev/null | jq -r .url > .planning/phases/05-deploy-vercel/05-DEPLOY-URL.txt
   ```
   If `jq` parsing fails, fall back to the simpler:
   ```bash
   vercel ls --prod | head -2 | tail -1 | awk '{print $2}' > .planning/phases/05-deploy-vercel/05-DEPLOY-URL.txt
   ```
   Or just paste the URL the deploy printed.

4. **Smoke-test the deployment with curl** (does NOT require browser — proves the server is up):
   ```bash
   DEPLOY_URL=$(cat .planning/phases/05-deploy-vercel/05-DEPLOY-URL.txt | tr -d '\n')
   curl -sS -o /dev/null -w '%{http_code}\n' "https://${DEPLOY_URL}/"
   ```
   Expect: `200`.

5. **Smoke-test /api/debrief on production** with the Take B fixture (proves DASHSCOPE_API_KEY works in prod):
   ```bash
   curl -sS -X POST "https://${DEPLOY_URL}/api/debrief" \
     -H 'content-type: application/json' \
     -d '{"transcript":[{"role":"user","text":"line 6 grid five-niner-niner-seven-niner-niner"},{"role":"pilot","text":"copy five-niner-niner-seven-niner-niner cleared hot"}],"impact":{"distanceToTarget":100.3,"distanceToFriendlies":19.7,"grid":"599799"},"correctGrid":"599699"}' \
     | bun -e "let d=''; for await (const c of process.stdin) d+=c; const j=JSON.parse(d); if(j.verdict!=='unsafe'||!j.critique||j.critique.length<10||j.critique.startsWith('Debrief generation failed')){console.error('FAIL',d);process.exit(1)}console.log('PROD_CURL_OK verdict='+j.verdict)"
   ```
   Expect: `PROD_CURL_OK verdict=unsafe`.

6. Commit the URL artifact:
   ```bash
   git add .planning/phases/05-deploy-vercel/05-DEPLOY-URL.txt
   git commit -m "chore(phase-05): record production deploy URL"
   ```
